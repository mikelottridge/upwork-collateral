# Psychologist Deck Feedback Summary

## Main Takeaways

The feedback is mostly good, and Claude independently validated the main themes.

The biggest improvements would be:

- move most of the deck into present tense
- lead more clearly with practitioner/patient communication, not just analysis
- reduce UI elements that look clickable, especially `Python / Flask`
- strengthen the closing sentence
- reframe the footer so it feels like transparent context, not a disclaimer

## What I Agree With

- Present tense is a real upgrade across the hero, slides, and narration.
- The deck should emphasize that the system helps a practitioner explain results and helps a patient understand them.
- The workflow language is stronger when it focuses on communication and finished deliverables, not just analysis.
- The charting / time-series point is strong and should stay prominent.
- The report slide should clearly say that charts, interpretation, and report structure stay together in one output path.
- The long-running engagement is an important proof point.

## What I Only Partly Agree With

- The strict `data` as plural rule is defensible, but I would not force it everywhere if it makes the copy sound stiff.
- `Analyses live inside the report` is fine, but the bigger fix is present tense, not necessarily pluralizing `analysis`.
- Some of the suggested rewrites are already very close to what the current deck says, so in a few places the feedback is more confirmation than correction.

## What I Disagree With

- I would not treat `data are` / `data do` as a universal rule for this kind of client-facing collateral.
- I do not think every single sentence must be present tense. The general shift is right, but it does not need to become mechanical.

## Specific Fixes Worth Making

- Rework the hero so it says the value is making results clear enough for practitioner explanation and patient understanding.
- Convert most past-tense slide copy and narration into present tense.
- Replace `Python / Flask` as a metric, since it reads more like a clickable tag than a meaningful result.
- Tighten slide 1 around the idea that the real problem is carrying assessment data forward into something usable and discussable.
- Update slide 2 so the workflow is described in present tense.
- Update slide 3 so it explicitly emphasizes time-series / longitudinal communication value.
- Update slide 4 so the report copy uses full present-tense sentences rather than fragments.
- Tighten slide 5 so the durability / trust proof reads more strongly.
- Replace the closing phrase `this is the kind of work Mike is built for` with something more direct.
- Reframe the footer as transparent context, for example: `Demonstrated with synthetic data. The focus is workflow structure and reporting output, not confidential client content.`

## Bottom Line

The deck direction is good. The biggest wins are present tense, stronger communication-first framing, cleaner metadata treatment, and a stronger close.
