## Short Description

I built a Python/Flask application for healthcare document processing, focused on analyzing lab-report PDFs and turning document-heavy inputs into structured, reviewable outputs. The workflow used staged AI orchestration rather than a single prompt, so intermediate steps could be checked before producing a final result. I also handled the application layer and deployment setup so the system could be used as a real tool rather than a demo.

For public sharing, this project should be shown with synthetic or redacted examples only, but it is a strong example of code-first AI document processing in a high-trust environment.

## Alternate Description

This project involved building a Python/Flask web application for lab-report PDF analysis and downstream treatment-review workflows. The system parsed medical documents, ran multiple staged AI steps, and produced structured outputs that were easier to review and act on. It was designed as a practical workflow tool, with deployment and operational usability in mind, not just a proof of concept.
